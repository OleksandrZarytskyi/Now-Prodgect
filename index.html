Base

  • git config --global user.name blabla ім'я того хто робить цей створив новий commit
  • git config --global user.email blabal@bla імейл толо хто сворює новий commit
  • git init - команда для створення архіву з версіями данних
  • git status - команда яка говорить про стан данних в каталозі
  • git add - збарігає данні(додати всі файли add.)
  • git commit - ім'я версії даних і доває його до архіву (commit -m"назва нового коміту"
  • git log - логи наших і історя commit
  • git diff - які зміни пройшли в файлах
  • git branch - дає можливість працювати/створювати нові гілки
  • git checkout - дає можливісь дивитись старі версії
  • git push - відправити нову версію на сервер
  • git pull - стягнути нову версію з серверу
  • git merge - Це обеднання двух версій коду в один branch
  • git clone - витягнути фали з github(копіює до нас окрему версію архіву)

Gitignore файл - це файли з якими ми не будемо надалі працювати(по причині готовності), їх зачасту скривають для комфортності подальшої роботи

Advanced

  • git revert - Видаляє(відкатує) останій commit зі всіми данними, але залишає в логах
  • git reset --hard log видяляє данні про логи, після вакзаного commit про більш нові
  • git rebase - Обєднання двух branch і робота по їх злиттю в автоматичному режимі
  • git cherry-pick - те саме rebase тільки в ручну
  • git stash - ховає - поки нам не потрібні файли підчас роботи в github
